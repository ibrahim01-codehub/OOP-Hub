OOP Concept

Methods

Methods are the functions that live inside a class. This page covers four decisions you'll make about them constantly: who can call a member (private vs public), whether your type defaults to open or closed (class vs struct), how an object's lifetime starts and ends (constructor vs destructor), and two special function types that bend the normal rules (friend vs virtual).

💡

These four pairs aren't really opposites — they're independent knobs. A class can be public-by-default (a struct) and still have a constructor, a destructor, and virtual methods.

What You'll Learn Here

  • How private and public control access to a member
  • The one real difference between class and struct in C++
  • How constructors and destructors manage an object's lifetime automatically
  • When to reach for a friend function versus a virtual function
Access Modifiers

Private vs Public

private members can only be reached from inside the class itself. public members can be reached from anywhere the object is visible. This is the mechanism encapsulation is built on: keep data private, expose a controlled public interface to work with it.

private

Accessible only inside the class's own methods. The default access level for a class.

public

Accessible from anywhere the object exists — other classes, other functions, main().

Syntax

syntax access-modifiers.cpp 
class ClassName {
private:
    int hiddenData;      // only usable inside this class
 
public:
    void setData(int value) { hiddenData = value; }
    int getData() { return hiddenData; }
};

Code Example (C++)

example bank-account.cpp 
#include <iostream>
using namespace std;
 
class BankAccount {
private:
    double balance;                 // can't be touched directly
 
public:
    void deposit(double amount) { balance += amount; }
    double getBalance() { return balance; }
};
 
int main() {
    BankAccount acc;
    acc.deposit(500);            // OK — public method
    // acc.balance = 999;       // ERROR — private, won't compile
    cout << "Balance: " << acc.getBalance() << endl;
    return 0;
}
Output 
Balance: 500
Type Declaration

Class vs Struct

In C++, class and struct can both hold attributes, methods, constructors, and take part in inheritance — the only real difference is their default access level.

class

Members are private by default. The conventional choice when a type has behavior to protect.

struct

Members are public by default. The conventional choice for simple, data-only bundles.

Syntax

syntax class-vs-struct.cpp 
class PointClass {
    int x, y;        // private by default
};
 
struct PointStruct {
    int x, y;        // public by default
};

Code Example (C++)

example point.cpp 
#include <iostream>
using namespace std;
 
struct Point {
    int x;
    int y;              // public — no getters needed
};
 
int main() {
    Point p;
    p.x = 4;                 // works directly — struct is public by default
    p.y = 9;
    cout << "(" << p.x << ", " << p.y << ")" << endl;
    return 0;
}
Output 
(4, 9)
Object Lifecycle

Constructor vs Destructor

A constructor runs automatically when an object is created, usually to set up its initial data. A destructor runs automatically when an object is destroyed, usually to release anything it was holding onto.

Constructor

Same name as the class, no return type. Can be overloaded to accept different parameters. Runs once, on creation.

Destructor

Class name prefixed with ~, no parameters, no return type, and only one per class. Runs once, on destruction.

Syntax

syntax constructor-destructor.cpp 
class ClassName {
public:
    ClassName() {           // constructor
        // runs when an object is created
    }
 
    ~ClassName() {          // destructor
        // runs when an object is destroyed
    }
};

Code Example (C++)

example logger.cpp 
#include <iostream>
using namespace std;
 
class Logger {
public:
    Logger() { cout << "Logger created" << endl; }
    ~Logger() { cout << "Logger destroyed" << endl; }
};
 
int main() {
    Logger log;           // constructor runs here
    cout << "Doing some work..." << endl;
    return 0;             // destructor runs here, automatically
}
Output 
Logger created
Doing some work...
Logger destroyed
Advanced Functions

Friend Function vs Virtual Function

A friend function is an outsider granted access to a class's private members — useful, but it opts out of encapsulation, so it's worth using sparingly. A virtual function is the opposite kind of tool: a member function a base class expects derived classes to override, resolved at runtime rather than compile time — the mechanism behind polymorphism.

friend

A non-member function or class explicitly given access to a class's private and protected members. Not inherited, not called through an object.

virtual

A member function marked so a derived class's override is called, even through a base-class pointer or reference. Resolved at runtime.

Syntax

syntax friend-virtual.cpp 
class ClassName {
    friend void outsideFunction(ClassName obj); // grants access
 
public:
    virtual void speak() {          // overridable by derived classes
        // default behavior
    }
};

Code Example (C++)

example animal.cpp 
#include <iostream>
using namespace std;
 
class Animal {
private:
    int age;
 
public:
    Animal(int a) : age(a) {}
 
    // friend: an outsider allowed to read private data from two Animals
    friend bool isOlder(Animal& a, Animal& b);
 
    // virtual: derived classes are expected to override this
    virtual void speak() { cout << "..." << endl; }
};
 
bool isOlder(Animal& a, Animal& b) {
    return a.age > b.age;    // reaching into private data directly
}
 
class Dog : public Animal {
public:
    Dog(int a) : Animal(a) {}
    void speak() override { cout << "Woof!" << endl; }
};
 
int main() {
    Animal generic(3);
    Dog rex(5);
 
    rex.speak();                                  // virtual: calls Dog's version
    cout << isOlder(rex, generic) << endl;        // friend: 1 (true)
    return 0;
}
Output 
Woof!

Pros of Knowing These Four

  • private/public gives you deliberate, controlled access to data
  • class/struct lets you pick the right default for the job, simple data vs protected behavior
  • Constructors/destructors automate setup and cleanup — no manual bookkeeping
  • friend/virtual extend the object model exactly where the rules need to bend

Real-Life Analogy

🏢

Think of an office building. private rooms need a keycard (only staff — the class itself — get in), public areas are open to visitors. A struct is like a small open-plan desk — no doors by default. The constructor is the building opening for the day, the destructor is locking up at night. A friend is a contractor handed a one-time keycard; a virtual role is a job title — "manager" — where each branch fills it with its own person.
