Encapsulation

Encapsulation is the practice of binding data and the methods that work on that data into a single unit, then restricting direct access to some of that data from outside the unit.

💡

It's one of the four pillars of OOP. It helps with data hiding and protects the integrity of an object's internal state.

What You'll Learn Here

  • Why raw, unrestricted access to data causes problems
  • How private attributes and public methods work together
  • How to write getters and setters
  • Where encapsulation fits among the other three pillars

Why Encapsulation?

  • Protects data from unintended interference and misuse
  • Hides internal implementation details from the outside world
  • Provides controlled access through public getters and setters
  • Improves maintainability and flexibility of code

How It Works

The data (attributes) are kept private, and access is provided only through public methods. Outside code can use those methods freely, but it can never reach in and touch the private data directly.

Class
Public Methods (getters / setters)
Private Data (attributes)
→ uses methods ✕ no direct access
Outside Code

Syntax

syntax encapsulation.cpp 
class ClassName {
private:
    int attribute;              // hidden
 
public:
    void setAttribute(int value) { attribute = value; }
    int getAttribute() { return attribute; }
};

Code Example (C++)

example car.cpp 
#include <iostream>
using namespace std;
 
class Car {
private:
    string brand;                 // private attribute
 
public:
    // public method to set brand
    void setBrand(string b) {
        brand = b;
    }
 
    // public method to get brand
    string getBrand() {
        return brand;
    }
};
 
int main() {
    Car myCar;
    myCar.setBrand("Toyota");
    cout << "Brand: " << myCar.getBrand() << endl;
    return 0;
}
Output 
Brand: Toyota

Pros of Encapsulation

  • Binds data and methods together
  • Data is hidden from outside access
  • Access is provided only through controlled public methods
  • Helps protect data and makes code more robust

Real-Life Analogy

💊

Think of a medicine capsule. You get the benefit of what's inside through the capsule itself (the public interface), but you can't see or change what's inside it directly (the private data).
