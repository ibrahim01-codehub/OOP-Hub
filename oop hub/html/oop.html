Definition

Object-Oriented Programming (OOP) is a programming paradigm that structures software around objects, which are instances of classes. Each object encapsulates data (attributes) and the methods (functions) that operate on that data. By organizing programs into self-contained, interacting objects, OOP improves code reusability, readability, maintainability, and scalability, making it one of the most widely adopted approaches for modern software development.

Why was oop created?
Early software was mostly written using Procedural Programming.
As software became larger, developers encountered several problems:
  • Difficult to maintain
  • Code Duplication
  • Poorly Organized code
  • Hard Debugging
  • Difficult Network
  • Low useability

Managing thousands of functions in a single project became increasingly complex.
Object-Oriented Programming was introduced to solve these challenges by organizing software into independent, reusable objects.

What exactly is Object Object-Oriented-Programming

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of objects.
An object combines:

  • Data (attributes)
  • Functions (methods)

into a single unit
Instead of separating variables and functions throughout a program, OOP groups related information together.

Main Idea

Everything should be represented as an object whenever possible.

Objects interact with one another by calling methods and exchanging information.

Advantages
  • Better code organization
  • Easier Maintaince
  • Improved Readibility
  • Code Reuseability
  • Higher Security Through Encapsulation
  • Simplifies Debugging
  • Easier Collaboration among developers
  • Better scalabilty
  • Easier Testing
  • Modular Software Develpoment
Disadvantages
  • Steeper learning curve – especially for beginners
  • Programs can become larger and slower (overhead)
  • Not suitable for all problems (e.g. small scripts)
  • Over-engineering – unnecessary abstraction
  • Increased complexity in design (UML, planning)
  • Harder to optimize performance in some cases
  • Memory consumption can be higher
  • More code to write compared to procedural
  • Can lead to deep inheritance hierarchies (difficult to maintain)
  • May require careful planning to avoid tight coupling

Despite these drawbacks, OOP remains a powerful tool for large‑scale, collaborative, and long‑term projects.
