Polymorphism

Polymorphism — literally "many forms" — is the ability for different classes to respond to the same method call in their own way. Code written against one shared interface can work with any class that implements it, without knowing exactly which one it's dealing with.

💡

C++ has two kinds: compile-time polymorphism (function/operator overloading) and runtime polymorphism (virtual functions). This page focuses on runtime polymorphism, since it's the one built directly on inheritance.

What You'll Learn Here

  • How a base-class pointer can call different overridden methods
  • The role virtual and override play in making this work
  • Why polymorphism depends on inheritance
  • How to loop over different objects through one shared type

Why Polymorphism?

  • One function call can behave differently depending on the object
  • New classes can be added without changing the code that uses them
  • Collections of related objects can be handled through one shared type
  • Keeps calling code simple even as the number of classes grows

How It Works

A base class marks a method virtual. Every derived class can then override it with its own version. Calling that method through a base-class pointer or reference always runs the derived class's version — decided at runtime, not compile time.

Syntax

syntax polymorphism.cpp 
class Base {
public:
    virtual void action() { /* default behavior */ }
};
 
class Derived : public Base {
public:
    void action() override { /* its own behavior */ }
};
 
Base* obj = new Derived();
obj->action();                // runs Derived's version

Code Example (C++)

example shapes.cpp 
#include <iostream>
using namespace std;
 
class Shape {
public:
    virtual void draw() { cout << "Drawing a shape" << endl; }
};
 
class Circle : public Shape {
public:
    void draw() override { cout << "Drawing a circle" << endl; }
};
 
class Square : public Shape {
public:
    void draw() override { cout << "Drawing a square" << endl; }
};
 
int main() {
    Shape* shapes[] = { new Circle(), new Square() };
 
    for (Shape* s : shapes) {
        s->draw();               // same call, different result each time
    }
    return 0;
}
Output 
Drawing a circle
Drawing a square

Pros of Polymorphism

  • Makes code flexible and easy to extend with new classes
  • Simplifies code that has to work with many related types
  • Supports dynamic, runtime-decided behavior
  • Encourages designing around interfaces rather than concrete types

Real-Life Analogy

🔘

Think of one "speak" button. Press it on a dog and it barks; press it on a cat and it meows. Same action requested, different result depending on which object receives it.
