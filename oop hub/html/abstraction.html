Abstraction

Abstraction means exposing only the essential features of something while hiding the complex details behind it. The user of a class gets a simple interface to work with, without needing to know how that interface is implemented underneath.

💡

Abstraction and encapsulation are often confused. Encapsulation hides data behind access modifiers; abstraction hides complexity behind a simple interface.

What You'll Learn Here

  • The difference between abstraction and encapsulation
  • How abstract classes and pure virtual functions define an interface
  • Why you can't create an object directly from an abstract class
  • How different classes can implement the same abstract interface

Why Abstraction?

  • Reduces complexity for whoever is using the class
  • Lets implementation details change without breaking callers
  • Forces a consistent interface across different implementations
  • Keeps the caller focused on what something does, not how

How It Works

In C++, abstraction is achieved with abstract classes: a class that declares one or more pure virtual functions — methods with no body, marked = 0. An abstract class can't be instantiated directly; each derived class must supply its own implementation of those methods.

What the Caller Sees

shape.area();

hides the details ↓
What's Hidden

Circle: π × r × r  ·  Rectangle: width × height

Syntax

syntax abstraction.cpp 
class AbstractClassName {
public:
    virtual void doSomething() = 0;   // pure virtual — no body
};
 
class Concrete : public AbstractClassName {
public:
    void doSomething() override {
        // real implementation lives here
    }
};

Code Example (C++)

example shape.cpp 
#include <iostream>
using namespace std;
 
class Shape {                       // abstract class
public:
    virtual double area() = 0;       // pure virtual — defines the interface
};
 
class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() override { return 3.14159 * radius * radius; }
};
 
int main() {
    Circle c(4);
    Shape& s = c;                     // interact through the abstract interface
    cout << "Area: " << s.area() << endl;
    return 0;
}
Output 
Area: 50.2654

Pros of Abstraction

  • Simplifies how a class is used from the outside
  • Internal implementation can change freely without breaking callers
  • Encourages designing around a clear, stable interface
  • Makes large systems easier to reason about in pieces

Real-Life Analogy

🚗

Think of driving a car. You use the steering wheel and pedals — the interface — without knowing exactly how the engine, fuel injection, or ignition timing work underneath. Those details are abstracted away.
