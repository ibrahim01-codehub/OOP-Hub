Classes & Objects

A class is a blueprint that defines what data and behavior something has. An object is one real, usable thing built from that blueprint. Every other OOP concept — encapsulation, inheritance, polymorphism — is built on top of this one pair of ideas.

💡

A class doesn't take up memory for its data until you create an object from it. The class is the plan; the object is the thing.

How It Works

A class groups related attributes (data) and methods (functions) under one name. Once the class exists, you can create as many objects from it as you need — each one holds its own copy of the attributes, but shares the same methods defined in the class.

Syntax

syntax class-and-object.cpp 
class ClassName {
public:
    // attributes
    int attributeName;
 
    // methods
    void methodName() {
        // behavior
    }
};
 
int main() {
    ClassName objectName;        // creating an object
    objectName.methodName();       // using an object
}

Code Example (C++)

example car.cpp 
#include <iostream>
using namespace std;
 
class Car {
public:
    string brand;
    int speed;
 
    void honk() {
        cout << brand << " says: Beep!" << endl;
    }
};
 
int main() {
    Car car1;              // object 1
    car1.brand = "Toyota";
    car1.speed = 120;
 
    Car car2;              // object 2, independent of car1
    car2.brand = "Honda";
    car2.speed = 140;
 
    car1.honk();
    car2.honk();
    return 0;
}
Output 
Toyota says: Beep!
Honda says: Beep!

Pros of Classes & Objects

  • One blueprint can produce unlimited independent instances
  • Related data and behavior live in one place, not scattered across the program
  • Real-world entities become easy to model directly in code
  • Forms the foundation every other OOP concept builds on

Real-Life Analogy

🍪

A class is a cookie cutter, an object is a cookie. The cutter defines the shape once — every cookie stamped from it follows that shape, but each cookie is still its own, separate cookie.
